# Custom Store

By default, Berry ORM will create IdentityMaps internally to store references to all the entities, and you need to manage your data yourself.

For a certain entity, if you want to share the store with Berry ORM, you can configure it in @Entity() options.

Â 





@Entity({ map: () => yourMap })
export class User extends BaseEntity<User, "id"> {
  @Field({ type: "primary" })
  id!: number;
}

TIP

The function passed will be invoked when instantiating BerryOrm.
